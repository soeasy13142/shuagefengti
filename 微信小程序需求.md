# 角色设定

你是一名资深全栈工程师兼产品设计师，请开发一个现代化移动端学习小程序。

在开始编码前，请先阅读完整需求，并严格按照需求拆分模块实现。不要擅自修改产品架构。

---

# 项目目标

这是一个以「Markdown题库导入 + 单题刷题」为核心功能的学习工具箱。

用户可以导入 Markdown 格式的题库文件，系统自动解析题目，然后以“一题一页”的方式进行刷题。

未来会不断增加各种学习工具，因此架构必须支持插件化扩展。

本项目的核心定位：

- 刷题平台
- 学习工具箱
- 后续可持续扩展

不要把项目设计成单纯的题库 App。

# Markdown解析规则

题目格式示例：

```markdown
# 第一章

## 1

什么是TCP三次握手？

A. xxxx

B. xxxx

C. xxxx

D. xxxx

答案：A

解析：
xxxxx
```

系统自动提取：

```json
{
  "id": "1",
  "question": "什么是TCP三次握手？",
  "type": "single_choice",
  "options": [
    "A. xxxx",
    "B. xxxx",
    "C. xxxx",
    "D. xxxx"
  ],
  "answer": "A",
  "analysis": "xxxxx"
}
```

请编写可扩展解析器。

后续支持：

- 单选题
- 多选题
- 判断题
- 简答题

因此解析器必须预留题型字段。

# 刷题模式

点击题库进入刷题。

采用：

**一题一页**

模式。

页面布局：

```text

题目内容

选项A
选项B
选项C
选项D

提交按钮
```

提交后显示：

- 正确答案
- 解析

然后出现：

- 下一题按钮
