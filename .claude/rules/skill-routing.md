# 技能路由（UI/UX/设计任务专用）

> 仅针对 **UI / 页面 / 交互 / 品牌 / 设计系统** 类任务。一般代码任务直接看 ECC 命令清单。

## 第一步：任务分类

| 类别 | 触发场景 |
|---|---|
| **UI/页面设计** | 新建页面、改 UI、布局、样式、配色、卡片、按钮、表单 |
| **UX/交互设计** | 用户流程、交互方案、信息架构、可用性、用户研究 |
| **品牌/设计系统** | 品牌标识、设计 token、Logo、企业 VI、设计规范 |
| **代码审查** | 改动完成后、PR review、找 bug、安全审计 |
| **文档/API 查询** | 库、框架、SDK、API、CLI 工具、配置、迁移 |
| **规划/架构** | 大功能规划、多文件改动、架构决策 |
| **纯逻辑/工具函数** | 工具函数、存储封装、解析器、算法 |
| **微信小程序页面** | 小程序页面开发（WXML/WXSS/JS） |

## 第二步：匹配技能

| 类别 | 首选技能 | 说明 |
|---|---|---|
| UI/页面设计 | `ui-ux-pro-max` | 本项目用暖奶油画布风格（claude-design） |
| UX/交互设计 | `ux-designer` | 用户研究和信息架构 |
| 品牌/设计系统 | `ckm-design-system` | 设计 token、组件规范 |
| 代码审查 | `code-reviewer` | 完成后必须调用 |
| 文档/API 查询 | `context7-mcp` | 即使认为知道也要查 |
| 规划/架构 | `superpowers:brainstorming` → `superpowers:writing-plans` | 先发散再落地 |
| 纯逻辑/工具函数 | 无特定技能 | 直接实现，完成后 code-review |
| 微信小程序页面 | `ui-ux-pro-max`（UI）+ 直接实现（逻辑） | 业务逻辑直接写 |

## 第三步：执行路由

```
□ 已完成任务分类
□ 已查表确定需要的技能
□ 如有匹配技能 → Skill 工具加载
□ 如无匹配技能 → 说明"未找到适用技能"
□ 已在回复开头说明使用的/未使用的原因
```

## 第四步：技能使用后收尾

- **UI 技能**：确保符合设计规范
- **`code-reviewer`**：每批代码改动后调用
- **`context7-mcp`**：先 resolve-library-id 再 query-docs
