## LL(1) 分析器约定（统一规格）

| 项 | 取值 |
|---|---|
| 文法输入格式 | 产生式列表，`A → α | β`，支持 ε 产生式（`ε` 或 `epsilon`）|
| 非终结符限制 | ≤ 20 |
| 终结符限制 | ≤ 20 |
| 不动点迭代轮数上限 | 100 轮 |
| 输入串结束符 | `$` |
| FIRST 集 | 含 ε（用 `'ε'` 字符串表示） |
| FOLLOW 集 | 不含 ε，含 `$`（start 符号的 FOLLOW） |
| 冲突检测 | 分析表同一格子 >1 条产生式 |
| 预设示例 | 3 个：表达式文法、括号匹配、if-else 文法 |

核心算法（不动点迭代）：
1. 初始化每个符号的 FIRST 为空集合
2. 终结符：FIRST[term] = { term }
3. 非终结符：遍历每个产生式 A → X1 X2 ... Xk
   - 将 FIRST[X1] - {ε} 加入 FIRST[A]
   - 如果 X1 能推出 ε，将 FIRST[X2] - {ε} 加入 FIRST[A]
   - 依此类推，直到遇到不能推出 ε 的符号
   - 如果所有 Xi 都能推出 ε，将 ε 加入 FIRST[A]
4. 重复直到所有集合不再变化（最多 100 轮）

- [ ] **Step 3: Implement `utils/ll1-table.js`**

核心算法：
1. 初始化 table[N][T] = null, conflicts[N][T] = []
2. 遍历每个产生式 A → α：
   - 对每个 a ∈ FIRST[α]（a ≠ ε）：table[A][a] = 该产生式
     - 若已有其他产生式 → 冲突记录
   - 如果 ε ∈ FIRST[α]：对每个 b ∈ FOLLOW[A]：table[A][b] = 该产生式
     - 若已有其他产生式 → 冲突记录
   - 如果 ε ∈ FIRST[α] 且 `$` ∈ FOLLOW[A]：table[A]['$'] = 该产生式
     - 若已有其他产生式 → 冲突记录
3. 返回 { table, conflicts }

Expected: FAIL with "Cannot find module '../../utils/ll1-parse'"

- [ ] **Step 3: Implement `utils/ll1-parse.js`**

核心算法：
1. 初始化栈 = ['$', startSymbol]，输入缓冲 = input[] + ['$']
2. 每一步记录当前栈快照、剩余输入、已产生的输出序列
3. 循环：
   - 栈顶为终结符或 $：与输入第一个符号比较
     - 匹配 → 弹出栈顶，消耗输入，记录 'match'
     - 不匹配 → 记录 'error'
   - 栈顶为非终结符：查 table[N][a]
     - 有产生式 → 弹出 N，将 rhs 逆序入栈，记录 'expand'
     - 无产生式 → 记录 'error'
   - 栈顶为 $ 且输入也为 $ → 'accept'
4. 返回步进列表

4. 非终结符用 `#cc785c`（主色），终结符用 `#141413`（暖墨色）
