## Regex→DFA 约定（统一规格）

| 项 | 取值 |
|---|---|
| 正则语法 | `*`（克林闭包）、`|`（选择）、`.`（连接隐式）、`()`（分组）、`?`（0或1）、`+`（1或多次）|
| NFA 构造 | Thompson 构造法，递归下降解析 regex |
| ε 转移 | 用 `'ε'` 作为转移 key |
| DFA 构造 | ε-闭包 → 子集构造法，直到无新状态 |
| DFA 状态命名 | 字母 'A', 'B', 'C'... 自动编号 |
| 递归深度上限 | 50 层（防止极端嵌套导致栈溢出） |
| DFA 状态数上限 | 50 个（防止渲染卡顿） |
| 输入字符集 | 仅 ASCII 可见字符 |
| 模拟运行 | 逐字符沿 DFA 转移，记录路径 |

核心函数：
- `parseRegex(regex)` — 递归下降解析器，返回 AST（支持 `*` `|` `.` `()` `?` `+`）
- `astToNFA(ast)` — Thompson 构造法，遍历 AST 递归构建 ε-NFA
- `regexToNFA(regex)` — 组合 parseRegex + astToNFA

实现细节：
- 递归下降解析：`parseExpr` / `parseTerm` / `parseFactor` / `parseAtom`
- Thompson 构造：每个 AST 节点对应一个 NFA 片段（`NFAFragment = { start, accept }`）
- 选择 `a|b`：新建起始/接受状态，ε 连接到两个子片段
- 连接 `ab`：第一个的接受连到第二个的起始（ε）
- 闭包 `a*`：新建起始/接受状态，ε 回路
- `a?`：等价于 `a|ε`
- `a+`：等价于 `aa*`
- 状态 ID 自增计数器

## Task 2: 子集构造法（ε-NFA → DFA）

**Files:**
- Create: `utils/nfa-dfa.js`
- Test: `tests/utils/nfa-dfa.test.js`

**Interfaces:**
- Consumes: NFA 结构（来自 regex-nfa.js）

  - `DFAState` 形状：`{ id: string, nfaStates: Set<number>, transitions: { [char]: dfaStateId }, isAccept: boolean }`

- [ ] **Step 2: Run test to verify it fails**

Run: `cd /Users/charliepan/Downloads/my-miniapp && npx jest tests/utils/nfa-dfa.test.js`
Expected: FAIL with "Cannot find module '../../utils/nfa-dfa'"

- [ ] **Step 3: Implement `utils/nfa-dfa.js`**

核心算法：
- `epsilonClosure`: BFS/DFS 从状态集合出发，沿着 ε 转移可到达的所有状态
- `move`: 从状态集合出发，沿着指定字符转移可到达的状态（不含 ε）
- `nfaToDFA`: 
  1. 初始 DFA 状态 = ε-闭包({nfa.start})
  2. 队列中取出未处理的 DFA 状态
  3. 对字母表中每个字符，计算 move → ε-闭包 → 已有则复用，否则新建
  4. 若 DFA 状态包含 NFA 接受状态，则 DFA 状态为接受
  5. 直到队列为空
