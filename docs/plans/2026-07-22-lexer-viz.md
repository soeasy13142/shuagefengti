| 项 | 取值 |
|---|---|
| 语言子集 | 类 C：关键字、标识符、数字、字符串、运算符、分隔符、注释 |
| 关键字 | `int`, `return`, `if`, `else`, `while`, `for`, `void`, `char`, `float`, `double`, `const`, `struct` |

| 预设示例 | 5 个：变量声明、函数调用、嵌套注释、多行字符串、完整程序 |

核心数据结构：
```js
// 规则优先级顺序（数组顺序决定匹配顺序）
TOKEN_RULES = [
  // 优先级 1: 关键字（优先于标识符）
  { name: 'KW_INT', pattern: /^int\b/, priority: 3, category: 'keyword' },
  { name: 'KW_RETURN', pattern: /^return\b/, priority: 3, category: 'keyword' },
  // ... 其他关键字
  // 优先级 2: 字符串常量
  { name: 'STRING', pattern: /^"(?:[^"\\]|\\.)*"/, priority: 2, category: 'literal' },
  // 优先级 3: 数字
  { name: 'NUMBER', pattern: /^\d+(?:\.\d+)?/, priority: 2, category: 'literal' },
  // 优先级 4: 运算符（含多字符 >=, <=, ==, !=, &&, ||）
  { name: 'OPERATOR', pattern: /^(?:>=|<=|==|!=|&&|\|\||[+\-*/%<>=!&|^~])/, priority: 2, category: 'operator' },
  // 优先级 5: 分隔符
  { name: 'SEP', pattern: /^[{}();,\[\]]/, priority: 2, category: 'separator' },
  // 优先级 6: 注释
  { name: 'COMMENT_LINE', pattern: /^\/\/[^\n]*/, priority: 1, category: 'comment' },
  { name: 'COMMENT_BLOCK', pattern: /^\/\*[\s\S]*?\*\//, priority: 1, category: 'comment' },
  // 优先级 7: 标识符（兜底）
  { name: 'ID', pattern: /^[a-zA-Z_]\w*/, priority: 1, category: 'identifier' },
]
```

实现细节：
- 规则按优先级降序排列（关键字 > 最长匹配 > 普通规则）
- 规则匹配用正则 `^` 开头确保从当前位置匹配
- `classifyToken(type)` 根据 type 前缀返回分类

Expected: FAIL with "Cannot find module '../../utils/lexer-engine'"

- [ ] **Step 3: Implement `utils/lexer-engine.js`**

核心算法：
1. 维护 `pos = 0`，`line = 1`，`col = 1`
2. 当 `pos < source.length`：
   - 跳过空白/换行（空格、制表符、换行符）
   - 遍历所有规则，找到匹配最长的规则（最长匹配原则）
   - 相同长度时，优先级高的规则胜出
   - 若匹配 → 创建 Token，记录步骤，pos += 匹配长度
   - 若不匹配 → 记录 LexError，pos++ 跳过非法字符
3. 返回 { tokens, errors, steps }

实现细节：
- 步骤记录包含 `sourceIndex`（光标位置），`token`（已识别的 Token），`buffer`（当前积累字符）
- 注释规则匹配后不产生 Token，直接跳过
- 字符串规则匹配整个字符串字面量为一个 Token
