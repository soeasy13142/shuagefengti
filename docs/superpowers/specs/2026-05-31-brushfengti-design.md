# 刷个冯题 — 微信小程序设计文档

## 概述

"刷个冯题"是一款多功能学习工具微信小程序。第一版聚焦**Markdown 试题导入与刷题**功能，主菜单预留多功能入口，后续可扩展排序可视化、单词记忆等工具。

**技术方案：** 纯本地存储（wx.setStorageSync），零后端成本，先跑通功能。

---

## 页面结构

```
首页（功能卡片入口）
  └── 刷题模块
        ├── 题目列表页（所有导入的试卷）
        │     ├── 导入试题（上传 .md）
        │     └── 点击某套题 → 选模式 → 刷题页
        ├── 答题记录页
        │     └── 记录详情 → 逐题回顾
        └── 错题本页
              └── 重做错题 → 刷题页
```

共 7 个页面。

---

## 模块 1：首页/主菜单

功能入口卡片列表（2 列网格）。

- 顶部：App 名称"刷个冯题"
- 卡片：图标 + 名称 + 简短描述
- "刷题"卡片可点击进入刷题模块
- 其他卡片置灰 + "即将上线"标签，点击弹 toast"功能开发中"

预留卡片：
- 刷题 — 导入试题，一题一题刷
- 排序可视化 — 即将上线
- 单词记忆 — 即将上线

## 模块 2：试题导入

**入口：** 题目列表页右上角"+"按钮

**上传流程：**
1. 调用 `wx.chooseMessageFile` 从聊天记录选择 `.md` 文件
2. `wx.getFileSystemManager().readFileSync` 读取内容
3. Markdown 解析器提取题目
4. 显示预览：共提取 X 题（单选 X、多选 X、判断 X、填空 X、简答 X）
5. 用户确认 → 保存到本地存储

**Markdown 解析策略（兼容两种格式）：**

- 题目：以数字编号开头（"1." 或 "## 第1题"）
- 选项：以字母开头（"A." / "A、" / "A)"）
- 答案：包含"答案"关键字的行
- 题型推断：
  - 有 A-D 选项 + 单个答案 → 单选
  - 有 A-D 选项 + 多个答案（"AB" 或 "A,C"）→ 多选
  - 只有 A/B 或 对/错 → 判断
  - 无选项 + 答案含填空标记（"___"）→ 填空
  - 无选项 + 答案是文本 → 简答

**存储结构：**

```javascript
// 试卷
{
  id: "uuid",
  name: "从文件名提取",
  importTime: "2026-05-31T...",
  questionCount: 50,
  questions: [
    {
      id: "q1",
      type: "single",           // single | multi | judge | fill | essay
      stem: "题目内容...",
      options: ["A. ...", "B. ...", "C. ...", "D. ..."],
      answer: "A",
      explanation: ""
    }
  ]
}
```

## 模块 3：刷题引擎

**进入方式：** 题目列表页点击某套题 → 弹出底部弹窗选模式 → 进入刷题

**两种模式：**
- 练习模式 — 答完立刻显示对错 + 正确答案 + 解析
- 考试模式 — 全部答完后统一批改

**单题页面布局：**
- 顶部：题号（第 X / Y 题）+ 题型标签
- 中间：题干区域（支持滚动）
- 选项区域（根据题型变化）：
  - 单选：radio 圆圈
  - 多选：checkbox 方框
  - 判断：两个大按钮"对"/"错"
  - 填空：输入框
  - 简答：多行文本框
- 解析区域（练习模式答后显示，考试模式隐藏）
- 底部按钮栏：上一题 / 提交 / 下一题

**各题型提交判分：**

| 题型 | 判分方式 |
|------|---------|
| 单选 | 答案比对 |
| 多选 | 全对才算对 |
| 判断 | 答案比对 |
| 填空 | trim 后比对，忽略大小写；多个空用 `___` 分隔，逐空独立判分，全对才算对 |
| 简答 | 显示参考答案，用户自行判断（标记"已作答"） |

**底部按钮逻辑：**

| 按钮 | 练习模式 | 考试模式 |
|------|---------|---------|
| 上一题 | 可回看，保留答题状态 | 可回看，可修改答案 |
| 提交 | 判分当前题 | 保存答案，不判分 |
| 下一题 | 解析看完后才可点 | 随时可点 |
| 最后一题 | 变成"查看结果" | 变成"交卷并查看结果" |

**结果页：**
- 正确率（百分比 + 环形进度图）
- 各题型正确数/总数
- 用时
- 错题列表（可点击进入重做）
- "重新刷题" / "返回首页"按钮

## 模块 4：答题记录

**存储结构：**

```javascript
{
  id: "record-uuid",
  paperId: "对应试卷 id",
  paperName: "试卷名称",
  mode: "practice",           // practice | exam
  startTime: "2026-05-31T...",
  endTime: "2026-05-31T...",
  duration: 1200,
  totalQuestions: 50,
  correctCount: 42,
  accuracy: 84,
  answers: {
    "q1": { userAnswer: "A", correct: true },
    "q2": { userAnswer: "BD", correct: false, correctAnswer: "BC" },
    "q3": { userAnswer: "快速排序", correct: true }
  }
}
```

**记录列表页：**
- 按时间倒序排列
- 每条显示：试卷名、模式标签（练习/考试）、正确率、用时、日期
- 点击进入详情 → 逐题回顾

**逐题回顾页：**
- 和刷题页面布局相同，但不可修改答案
- 每题标注：你的答案、正确答案、是否正确
- 错题有醒目标记（红色）

## 模块 5：错题本

**存储结构：**

```javascript
// wrongQuestions 存储
[
  {
    questionId: "q1",
    paperId: "试卷 id",
    question: { /* 完整题目对象 */ },
    wrongCount: 2,
    mastered: false,
    lastWrongTime: "2026-05-31T..."
  }
]
```

**数据来源：** 从所有答题记录中自动汇总错题

**错题列表页：**
- 支持按试卷分组 / 按时间排列（可切换）
- 每条显示：题干摘要（前 30 字）、题型标签、错误次数
- 右上角"重做错题"按钮

**重做错题：**
- 错题组成临时试卷，进入刷题引擎
- 答对后标记"已掌握"

**错题详情：**
- 题干完整内容
- 错误答案（红色）+ 正确答案（绿色）
- 解析（如果有）
- "标记已掌握"按钮

---

## 技术要点

- 存储：`wx.setStorageSync` 本地存储，key 命名规范：`papers`、`records`、`wrongQuestions`
- 文件上传：`wx.chooseMessageFile` 选择 .md 文件
- 文件读取：`wx.getFileSystemManager().readFileSync`
- Markdown 解析：纯正则实现，不引入外部库
- 扩展性：首页卡片数据驱动，新增功能只需加卡片配置 + 新页面
