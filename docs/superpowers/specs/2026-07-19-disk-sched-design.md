# 磁盘调度可视化 · 设计文档

> 日期：2026-07-19

> 状态：待用户审核

## 1. 目标

为「刷个冯题」小程序新增一个 **磁盘调度算法** 可视化教学页面：用户输入柱面号序列，可选 SCAN / C-SCAN / LOOK / C-LOOK 算法，观察磁头移动路径与总寻道时间，帮助 OS 学习者直观理解各调度算法的行为差异。

## 2. 范围

包含：
- 4 个磁盘调度算法：SCAN（电梯算法）/ C-SCAN / LOOK / C-LOOK
- 磁道序列输入：手动输入柱面号列表 + 🎲 随机生成
- 磁头起点配置：自定义起始位置 + 方向选择（向大号 / 向小号）
- 磁头移动动画：横向磁道轴，磁头从左到右 / 从右到左移动
- 性能指标：总寻道长度、平均寻道长度、磁头移动路径列表

不包含（明确不做）：
- SSTF（最短寻道时间优先，预留扩展）
- 多磁头 / RAID 场景
- 旋转延迟和传输时间的计算（仅关注寻道时间）
- 磁盘臂粘着（Arm Stickiness）模拟

## 4. 核心交互

```
┌─ 配置 ─────────────────────────────────────────┐
│  磁头起点: 50  方向: [← 小号] [大号 →]  🎲随机  │
│  柱面序列: 98, 183, 37, 122, 14, 124, 65, 67   │
│  [SCAN] [C-SCAN] [LOOK] [C-LOOK]               │
├─ 磁头移动 ─────────────────────────────────────┤
│  0────50───100───150───200───                   │
│  → → → → → → → ← ← ← ← ← → → → →             │
│  ●──────────────────────────────────────●       │
│  起点 50  98  183  122  37  14  65  67  124 终点│
├─ 性能指标 ─────────────────────────────────────┤
│  总寻道长度: 640                                  │
│  平均寻道长度: 80.0                              │
│  移动路径: 50→98→183→37→...                      │
│  算法对比: SCAN vs C-SCAN → C-SCAN 短 12% ▾      │
└─────────────────────────────────────────────────┘
```

用户配置磁头起始位置和方向，输入柱面号序列，选择算法后点击 ▶ 播放。磁头在横向磁道轴上以动画方式从左到右 / 从右到左移动，依次访问目标柱面。每个请求的柱面在轴上用标记点表示，访问过的变为绿色。底部显示实时累计的寻道长度和完整的磁头移动路径。

底部有算法对比区域，选中两个算法后可一键切换对比总寻道长度的差异。

## 6. 错误处理

| 场景 | 处理 |
|---|---|
| 柱面序列为空 | 禁用 ▶ 按钮，提示 "请添加柱面号" |
| 柱面号 < 0 | input 校验，提示 "柱面号不能为负" |
| 磁头起点 < 0 | 起点 input 校验，自动修正为 0 |
| 柱面号超出 0~199 范围 | 警告"超出常见磁盘柱面范围 0~199"，仍允许执行 |
| 动画中切换算法或方向 | 提示 "请先重置" |
| 随机生成超出范围 | 柱面号限制在 0~199 内 |

```js
// LOOK(up) 同输入
// 期望路径: 50→98→122→183→(转向)→37→14
// 注意：不到 199

// C-LOOK(up) 同输入
// 期望路径: 50→98→122→183→(跳回14)→37
// 注意：不到 199 也不到 0
```

## 8. 实施注意事项

1. **风格统一**：完全遵循 Claude Design 暖奶油画布（`#faf9f5` 背景 / `#efe9de` 卡片 / `#cc785c` CTA / Georgia 衬线标题）。
2. **纯函数优先**：`disk-scheduling.js` 全部无副作用。
3. **磁道轴可视化**：使用横向 `<scroll-view>` + 绝对定位标记点，磁头用三角形指示符。柱面范围默认 0~199。
4. **动画**：磁头移动用 WXSS `transition: left 300ms ease`，每次寻道逐步更新位置。
5. **请求数量上限**：最多 15 个柱面请求（可视化清晰度）。
6. **更新 PROJECT_HANDOFF**：完成时追加变更记录。

## 9. 风险与未来工作

| 风险 | 缓解 |
|---|---|
| 磁道范围 0~199 在窄屏上渲染密集 | 使用可缩放的时间轴，仅展示请求覆盖范围内的柱面刻度 |
| 大跨跳动画效果不自然（C-SCAN/C-LOOK 跳回对侧） | 跳回时用虚线动画 + 文字标注 "跳回" |
| 四种算法结果混淆 | 每个算法路径用不同颜色，对比区域并列显示 2 个算法的路径 |

未来可拓展：
- SSTF（最短寻道时间优先）
- 寻道时间 vs 旋转延迟的联合分析
- 多磁盘臂场景
- 实时输入序列的在线调度
