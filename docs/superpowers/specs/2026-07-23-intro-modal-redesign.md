# intro-modal 重设计

## 概述

重写 `intro-modal` 组件的 WXML + WXSS，解决当前弹窗字小、排版稀疏、易读性低的三大问题。保持数据结构不变（intro 配置不改），纯视觉层面翻新。

## 变更范围

仅改 `components/intro-modal/` 下的 3 个文件：
- `intro-modal.wxml` — 重写模板结构
- `intro-modal.wxss` — 完全重写样式
- `intro-modal.js` — 无变化

所有调用方（首页、tools-all、各工具页面）无需改代码。

## 布局结构

遮罩层 + 居中卡片，卡片区域改为两段式：

```
┌──────────────────────────────┐
│  头部区（工具名 + 元信息）    │
│                              │
│  内容区（卡片列表，可滚动）   │
│  ┌──┐                       │
│  │  │ 核心价值                │
│  └──┘                       │
│  ┌──┐                       │
│  │  │ 功能列表                │
│  └──┘                       │
│  ┌──┐                       │
│  │  │ 前置知识                │
│  └──┘                       │
│  ┌──┐                       │
│  │  │ 适合场景                │
│  └──┘                       │
│                              │
│  CTA 按钮                     │
└──────────────────────────────┘
```

## 弹窗卡片尺寸

卡片宽度保持 560rpx，最大高度 85vh。头部区不滚动，内容区滚动（max-height 限制）。

## 卡片区块

每个信息区块渲染为一张独立卡片：

- **背景**: #faf9f5，圆角 16rpx，内边距 24rpx
- **左侧色条**: 4rpx 宽 #cc785c 竖条标识区块类型
- **卡片标题**: 22rpx semibold，暖墨色，卡片内顶部
- **卡片间距**: 24rpx

### 核心价值卡片

显示 `intro.valueProp`。纯段落文字，26rpx，行高 1.7。

### 功能列表卡片

显示 `intro.features[]`。每项以 `•` 开头，主色 #cc785c，列表项之间 16rpx 间距。

### 前置知识卡片

显示 `intro.prerequisites`。纯段落文字，26rpx，行高 1.7。如果内容很短则单行展示。

### 适合场景卡片

显示 `intro.useCases[]`。每项以 `▸` 开头（语义与功能区分），主色 #cc785c，行高 1.8。

## 头部区

工具名和 tagline 不再放在滚动体内：

- **工具名**: 36rpx Georgia 衬线，字重 700，暖墨色
- **tagline**: 26rpx，正文色，下方 12rpx 间距
- **难度与标签**: 显示为圆角 chips，20rpx
  - 难度星级（如 `★★★☆☆ 中级`），来自组件 data 中的 `difficultyStars` + `difficultyLabel`（已由 `_loadToolData` 从 registry 计算）
  - 标签 chip（如 `#可视化` `#面试必考`），来自 `tool.tags` 数组

## 底部

- **CTA 按钮**: 100% 宽度，88rpx 高度，#cc785c 底色，圆角 24rpx，文字"开始体验 →"
- **按钮悬停**: #a9583e
- **底部提示**: "下次进入将直接打开工具"，20rpx，次要灰

## 动画保持

遮罩淡入淡出（opacity 200ms）、卡片缩放弹出（transform 300ms cubic-bezier）保持不变。

## 完成标准

- 所有字号 >= 20rpx
- 每个信息区块视觉可快速区分
- 在 320px-414px 宽度手机上可读
- 不破坏现有 js 逻辑
- 所有已有调用方无需修改
