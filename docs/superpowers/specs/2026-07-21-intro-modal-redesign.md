# 使用说明弹窗重设计（向导式分步）

## 目标

将 `intro-modal` 组件从纯文字弹窗改为向导式分步引导，提升正式感和用户体验。

## 结构

3 步向导，底部导航 + 进度点：

```
┌────────────────────────────────────┐
│                            ✕       │
│                                    │
│              📖                     │  ← 步骤大图标 (64rpx)
│                                    │
│       使用说明 · 概述               │  ← Georgia 32rpx 标题
│                                    │
│       ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐    │  ← 白色内容卡 #faf9f5
│       死锁模拟器是一个交互式教学      │
│       工具，帮助你直观理解死锁        │
│       的两种经典分析手段。           │
│       └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘    │
│                                    │
│        ●  ○  ○         第 1/3 步   │  ← 进度指示
│                                    │
│      [上一步]      [下一步 →]        │  ← 底部双按钮
│                                    │
└────────────────────────────────────┘
```

## 分步内容

### 步骤 1 · 概述

| 字段 | 值 |
|------|-----|
| 图标 | 📖 |
| 标题 | 使用说明 · 概述 |
| 正文 | 死锁模拟器是一个交互式教学工具，帮助你直观理解死锁的两种经典分析手段。通过资源分配图和银行家算法，你可以亲手构建死锁场景并观察检测过程。 |

### 步骤 2 · 操作指南

| 字段 | 值 |
|------|-----|
| 图标 | 🔧 |
| 标题 | 使用说明 · 操作指南 |
| 正文 | **资源分配图（RAG）**：点击「+进程」「+资源」添加节点（上限各 5 个）。先选择连线类型，再依次点击两个节点建立连线。点击「检测死锁」自动判断当前状态。支持拖拽节点调整布局。内置 3 种预设场景。 |
| 续行 | **银行家算法**：调整进程数和资源类型数，自动生成输入矩阵。在 Max 和 Allocation 单元格中输入数值，Need 矩阵自动计算。点击「检测安全性」运行算法，查看安全序列和步骤追踪。内置 3 种预设场景。 |

### 步骤 3 · 结果解读

| 字段 | 值 |
|------|-----|
| 图标 | 📊 |
| 标题 | 使用说明 · 结果解读 |
| 正文 | **安全状态**：显示安全序列，所有进程可顺利完成。 |
| 要点 | **死锁 / 不安全**：红色标记死锁进程，显示环路路径。 |
| 要点 | **步骤追踪**：`[满足]` 表示条件成立，`[不满足]` 表示条件不成立。 |

### 进度指示

三个圆点 + 计数文本，位于内容卡下方：

| 属性 | 值 |
|------|-----|
| 圆点尺寸 | 16rpx |
| 圆点间距 | 16rpx |
| 当前圆点 | `●` `#cc785c` |
| 未到圆点 | `○` `#ddd7cc` |
| 计数文本 | 24rpx，`#6c6a64`，`第 N/3 步` |
| 布局 | 居中，`display:flex`，居中对齐 |

### 底部导航

双按钮并排，间距 24rpx：

| 属性 | 值 |
|------|-----|
| 按钮高度 | 72rpx |
| 圆角 | 16rpx |
| 字号 | 28rpx |
| `上一步` 背景 | `#efe9de` → 点击 `#ddd7cc` |
| `上一步` 文字 | `#6c6a64` |
| `下一步` 背景 | `#cc785c` → 点击 `#a9583e` |
| `下一步` 文字 | `#faf9f5` |
| `开始使用` | 同 `下一步` 样式（第 3 步替换文字） |

### 按钮逻辑

| 步骤 | 上一步 | 下一步/开始使用 |
|------|--------|----------------|
| 1/3 | 置灰不可点击 (`opacity: 0.4`) | 「下一步 →」 |
| 2/3 | 正常 | 「下一步 →」 |
| 3/3 | 正常 | 「开始使用 ✓」 |

### 切换动画

使用 WXML `wx:if` + CSS transition：

| 切步方向 | 旧内容 | 新内容 |
|---------|--------|--------|
| 下一步 | slideOutLeft (0.2s) | slideInRight (0.25s) |
| 上一步 | slideOutRight (0.2s) | slideInLeft (0.25s) |

```
@keyframes slideInRight  { from { transform: translateX(60rpx); opacity: 0; } to { transform: translateX(0); opacity: 1; } }
@keyframes slideOutLeft  { from { transform: translateX(0); opacity: 1; } to { transform: translateX(-60rpx); opacity: 0; } }
@keyframes slideInLeft   { from { transform: translateX(-60rpx); opacity: 0; } to { transform: translateX(0); opacity: 1; } }
@keyframes slideOutRight { from { transform: translateX(0); opacity: 1; } to { transform: translateX(60rpx); opacity: 0; } }
```

### 边界状态

| 场景 | 处理 |
|------|------|
| 第 1 步点「上一步」 | 不可用，按钮置灰 |
| 第 3 步点「下一步」 | 触发 `onClose`，标记已读 |
| 关闭弹窗 | 上步动画中断，隐藏弹窗 |
| 快速连点切步 | 以最后一步为准，跳过中间态动画 |

## 涉及文件

| 文件 | 说明 |
|------|------|
| `components/intro-modal/intro-modal.wxml` | 改为分步结构，含图标/标题/内容卡/进度/按钮 |
| `components/intro-modal/intro-modal.wxss` | 新增分步布局、动画、进度点等全部样式 |
| `components/intro-modal/intro-modal.js` | 新增 `currentStep` data，`onPrev`/`onNext` 方法，动画状态管理 |
| `pages/deadlock/deadlock.js` | `DEADLOCK_INTRO` 改为 3 个分段的数组或对象；`showIntro` 内重置 step |
| `components/intro-modal/intro-modal.json` | 无需变更 |

## 数据模型

`introContent` 属性由字符串改为对象数组：

```js
// 调用处（deadlock.js）
const DEADLOCK_STEPS = [
  {
    icon: '📖',
    title: '使用说明 · 概述',
    body: '死锁模拟器是一个交互式教学工具...'
  },
  {
    icon: '🔧',
    title: '使用说明 · 操作指南',
    body: '资源分配图（RAG）...\n\n银行家算法...'
  },
  {
    icon: '📊',
    title: '使用说明 · 结果解读',
    body: '安全状态：...\n\n死锁/不安全：...\n\n步骤追踪：...'
  }
]
```

## 验证

- `npm test` 全绿
- 首次进入死锁页面 → 自动弹出，显示第 1 步
- 第 1 步「上一步」置灰
- 第 2 步「上一步」「下一步」都可用
- 第 3 步「下一步」变为「开始使用」
- 全程可点 ✕ 或蒙版关闭
- 关闭后标记已读，下次不再自动弹出
