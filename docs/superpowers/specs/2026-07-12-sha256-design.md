## 1. 目标

为「刷个冯题」小程序新增一个 **SHA-256 哈希算法** 的可视化教学页面：用户输入任意文本，可看到 64 轮压缩函数的完整 trace（W[0..63] 消息扩展 + a-h 8 个寄存器状态），并自动展示雪崩效应（1-bit 差异输入导致 ~50% 输出差异）。帮助密码学 / 信息安全学习者深入理解 SHA-256 的内部机制。

## 2. 范围

包含：
- 完整 SHA-256 实现（FIPS 180-4 §6.2 兼容，BigInt 模拟 32-bit 加法）
- 64 轮 trace 记录（每轮的 W[0..63] + a-h 寄存器 + T1/T2 中间值）
- 任意文本输入（UTF-8 编码）
- 步进控制：单轮前进 / 后退 / 跳轮 / 连续播放（速度 0.5x/1x/2x）
- 雪崩对比：自动生成 1-bit 差异输入，显示 hash 逐位差异
- 已知测试向量验证（FIPS 180-4 附录 B）

不包含（明确不做）：
- 其他哈希算法（MD5 / SHA-1 / SHA-3 / BLAKE 等）
- HMAC / PBKDF2 / bcrypt 等密钥派生函数
- 文件输入（仅文本输入，第一版）
- 二进制 / 十六进制输入
- GPU / WebAssembly 加速（性能足够，纯 JS 实现）

### 4.4 左视图：消息扩展 W[0..63]

- 64 个格子（4×16 网格）
- 当前轮的 Wt 高亮（边框加粗 + 背景色变化）
- 显示当前 Wt 计算公式（σ0 / σ1 / +）

### 4.5 右视图：压缩函数 a-h 寄存器

```text
┌────────────────────────────┐
│  a: 0x6a09e667  →  0x5d6aebb1│

│  h: 0x5be0cd19  →  0x78d0e6f4│

└────────────────────────────┘
```

## 5. 数据流

```text
用户输入 message
  ↓
sha256-trace.trace(message, encoding='utf-8')
  ├─ utf8Encode(message) → bytes[]
  ├─ padding：append 0x80 + 0x00... + 64-bit big-endian length
  ├─ split into 512-bit blocks
  ├─ 初始化 H[0..7] = FIPS 180-4 §5.3.3 常量
  ├─ 对每个 block 跑 64 轮：
  │    ├─ 生成 W[0..15] = block[16 × 32-bit words]
  │    ├─ 生成 W[16..63]：
  │    │    W[t] = σ1(W[t-2]) + W[t-7] + σ0(W[t-15]) + W[t-16]
  │    ├─ a..h = H[0..7] 当前值
  │    ├─ 64 轮迭代：
  │    │    T1 = h + Σ1(e) + Ch(e,f,g) + K[t] + W[t]
  │    │    T2 = Σ0(a) + Maj(a,b,c)
  │    │    h=g; g=f; f=e; e=d+T1; d=c; c=b; b=a; a=T1
  │    └─ 记录 RoundTrace = { W, a..h, T1, T2, K[t] }
  ├─ H[0..7] += a..h
  └─ 返回 { rounds: RoundTrace[64], finalHash }
  ↓
页面渲染 round 0 状态（初始 H）
  ↓
步进控制器推进 currentRound
  ↓
每步渲染：
  ├─ W 网格当前 Wt 高亮
  ├─ a-h 寄存器数值变化
  ├─ T1 / T2 当前计算式
  └─ 当前 round 进度
  ↓
雪崩对比卡：
  ├─ 自动生成 flipMessage（翻转第 1 bit）
  ├─ 计算 sha256(message) vs sha256(flipMessage)
  ├─ 逐位 XOR → 差异位数组
  └─ 显示汉明距离 + 可视化
```

`RoundTrace` 数据结构：

```js
{
  round: number,        // 1-64
  W: Uint32Array[64],   // 消息扩展
  a: number, b: number, c: number, d: number,
  e: number, f: number, g: number, h: number,
  T1: number,
  T2: number,
  K: number             // 当前轮的常量
}
```

## 6. 错误处理

| 场景 | 处理 |
|---|---|
| 输入 > 100KB | toast "输入过大（>100KB），请缩小"，**不**开始计算（避免 trace 卡顿） |
| 空输入 | 合法处理（已知 hash `e3b0c44298fc1c44...`） |
| 输入含非 UTF-8 字节 | TextEncoder 替换为 `?`（U+003F） |
| 32-bit 加法溢出 | `(x + y) >>> 0` 强制回卷 |
| 用户在 trace 计算完成前点击 ▶ | 禁用按钮，显示 "计算中..." |
| 跳转到非法轮号（<0 或 >63） | 输入校验，clamp 到 [0, 63] |

## 7. 测试

| 测试文件 | 覆盖 |
|---|---|
| `tests/utils/sha256.test.js` | FIPS 180-4 附录 B 已知向量：`""` / `"abc"` / `"abcdbcdecdefdefgefghfghighijhijkijkljklmklmnlmnomnopnopq"` / 长度边界 55/56/119/120 字节 |

| `tests/utils/sha256-avalanche.test.js` | 1-bit 差异 → 汉明距离在 100-156 之间（期望 128 ± 28） |

**关键测试用例**：

```js
// FIPS 180-4 附录 B
expect(sha256('', 'utf-8')).toBe(
  'e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855'
)
expect(sha256('abc', 'utf-8')).toBe(
  'ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad'
)

// 雪崩
const a = sha256('hello', 'utf-8')
const b = sha256('hellp', 'utf-8')  // 翻转 1 bit (o → p)
const diff = hammingDistance(a, b)
expect(diff).toBeGreaterThan(80)
expect(diff).toBeLessThan(180)
```

覆盖率目标 ≥ 80%。

## 8. 实施注意事项

1. **风格统一**：完全遵循 Claude Design 暖奶油画布。
2. **纯函数优先**：`sha256.js` / `sha256-trace.js` / `sha256-avalanche.js` 全部无副作用。
3. **BigInt 不用**：32-bit 加法用 `(x + y) >>> 0` 强制回卷（JavaScript 数字 53 位精度足够，节省性能）。
4. **已知向量优先**：实现完成后立即跑 FIPS 180-4 附录 B 测试，失败立刻排查。
5. **trace 数据结构**：每轮 trace 必须包含完整的 W + a-h + T1 + T2，便于后续教学扩展。
6. **性能**：100KB 输入的 trace 计算应在 1 秒内完成（64 轮 × N blocks，纯 JS 实现足够）。
7. **更新 PROJECT_HANDOFF**：完成时追加变更记录。

## 9. 风险与未来工作

| 风险 | 缓解 |
|---|---|
| 32-bit 算术溢出错误 | 严格 `(x + y) >>> 0` 强制回卷；FIPS 向量测试 |
| trace 数据量过大（长输入） | 限制输入 ≤ 100KB |
| 雪崩效应不直观 | 自动生成 1-bit 差异输入 + 逐位标红 |
| 教学深度 vs 性能 | 仅 64 轮全部展开，不展开消息扩展的递归层级 |

未来可拓展：
- HMAC / PBKDF2 派生
- 二进制 / 文件输入
- 多 hash 对比（同时展示 SHA-1 / SHA-256 / SHA-512）
- Merkle-Damgård 结构动画（多 block 拼接）
- 与 TLS-viz 集成（HMAC in TLS）
