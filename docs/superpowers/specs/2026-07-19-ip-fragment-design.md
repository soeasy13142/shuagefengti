# IP 分片可视化 · 设计文档

> 日期：2026-07-19
> 阶段：brainstorming → spec
> 状态：待用户审核
> 关联：`utils/tool-registry.js`（`ip-fragment.available = false` → `true`）、`app.json`

## 1. 目标

为「刷个冯题」小程序新增一个 **IP 分片与重组** 的可视化教学页面：用户调整原始报文大小与 MTU 参数，直观看到 IP 层如何将超大数据报切分为多个分片（含标识符、标志位、片偏移三字段的变化），以及接收端如何根据分片偏移量重组。帮助 CS 学习者理解 IP 层的分片机制与 MTU 对传输的影响。

## 2. 范围

包含：
- 自定义原始报文大小（100 ~ 10000 字节滑块）
- 自定义 MTU 值（500 ~ 2000 字节，默认 1500）
- 分片列表动画（每片显示：标识符 ID / MF 标志 / 片偏移 / 数据长度 / 首字节偏移）
- 分片偏移量计算过程展示（整数分片 vs 剩余字节处理）
- 反转重组动画：从最后一个分片开始合并回完整数据报
- 边界值测试按钮（如 MTU=500, 数据正好 500 时只有 1 片）

不包含（明确不做）：
- IPv6 分片（IPv6 依赖 Path MTU Discovery，分片行为不同）
- DF 标志（Don't Fragment）导致丢包的 ICMP 模拟
- 分片丢失→超时重组的超时机制
- 真实网络数据抓包输入

## 4. 核心交互

```
┌──────────────────────────────────────────────────────────────┐
│  报文大小: [███████████░░░░] 1500 字节   ← 滑块              │
│  MTU:      [███████░░░░░░░] 1000 字节   ← 滑块              │
│  头部: 20 字节  可用载荷: 980 字节/片                       │
│  [▶ 分片]  [↺ 重组回放]                                      │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  原始报文 (1500 字节)                                        │
│  ┌────────────────────────────────────────────────┐          │
│  │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│          │
│  └────────────────────────────────────────────────┘          │
│                                                              │
│  ┌── 分片 1 ──┐ ┌── 分片 2 ──┐ ┌── 分片 3 ──┐              │
│  │ [0..979]   │ │ [980..1959]│ │ [1960..]   │               │
│  │ ID=0x4a3b  │ │ ID=0x4a3b  │ │ ID=0x4a3b  │               │
│  │ MF=1, 偏移0│ │ MF=1, 偏移 │ │ MF=0, 偏移 │              │
│  │            │ │ =122       │ │ =245       │               │
│  └────────────┘ └────────────┘ └────────────┘               │
│                                                              │
│  --- 偏移量计算过程 ---                                      │
│  每片可携带最多 980 字节（MTU 1000 - IP头 20）              │
│  分片 1: 字节 0 ~ 979  → 偏移 0        (0 / 8)              │
│  分片 2: 字节 980 ~ 1959 → 偏移 122   (980 / 8)             │
│  分片 3: 字节 1960 ~ 1499 → 偏移 245  (1960 / 8)            │
│                                                              │
│  --- 关键字段 ---                                            │
│  标识符 (ID) = 0x4a3b  ← 三片相同                           │
│  标志 (Flags): MF=1 (还有分片) / MF=0 (最后分片)            │
│  片偏移 = (起始字节 ÷ 8)                                     │
└──────────────────────────────────────────────────────────────┘
```

用户操作流程：
1. 拖动两个滑块分别调整报文大小与 MTU
2. 点击「分片」，下方展示分片列表（每片显示 ID/MF/偏移/数据范围）
3. 展开偏移量计算过程的可折叠面板
4. 点击「重组回放」动画展示接收端按偏移量合并的过程

## 5. 数据模型 / 核心逻辑

```js
// 分片输出
{
  totalBytes: 1500,
  headerSize: 20,
  mtu: 1000,
  payloadPerFragment: 980,
  fragments: [
    {
      index: 1,
      dataStart: 0,
      dataEnd: 979,
      dataLength: 980,
      id: '0x4a3b',
      mf: true,           // 还有后续分片
      offset: 0,          // 片偏移（字节偏移 ÷ 8）
      offsetBytes: 0,     // 实际字节偏移
      explanation: '分片 1：字节 0~979（980 字节）'
    },
    {
      index: 2,
      dataStart: 980,
      dataEnd: 1959,
      dataLength: 980,
      id: '0x4a3b',
      mf: true,
      offset: 122,        // 980 / 8 = 122.5 → 122（整数除法）
      offsetBytes: 976,   // 122 * 8 = 976
      note: '偏移 122 × 8 = 976，实际首字节 980，间隙 4 字节',
      explanation: '分片 2：字节 980~1959（980 字节）'
    },
    {
      index: 3,
      dataStart: 1960,
      dataEnd: 1499,
      dataLength: 540,
      id: '0x4a3b',
      mf: false,
      offset: 245,        // 1960 / 8 = 245
      explanation: '分片 3（最后）：字节 1960~1499（540 字节）'
    }
  ],
  totalFragments: 3
}

// 核心算法
function fragment(datagramSize, mtu) {
  const HEADER = 20;
  const mtuPayload = mtu - HEADER;   // 每片最大数据
  const count = Math.ceil(datagramSize / mtuPayload);
  // 各片: 前 count-1 片 = mtuPayload 字节，最后一片 = 剩余
  // 偏移量 = 起始字节 / 8（IP 协议要求 8 字节对齐）
}
```

## 6. 错误处理

| 场景 | 处理 |
|---|---|
| 报文大小 ≤ MTU | 提示 "无需分片（报文不超过 MTU）"，显示单片 |
| 报文大小 > 65535 | clamp 到 65535（IP 报文中 Total Length 上限） |
| MTU < 68（IPv4 最小 MTU） | clamp 到 68 + toast "MTU 不得小于 68" |
| 滑块拖动过快导致频繁计算 | debounce 300ms |
| 不可分片场景（DF=1） | 未来扩展，当前版本不做 |
