# 首页卡片简化设计

> 2026-07-19 · 基于 feature/card-redesign 分支

## 1. 背景

当前首页卡片（2026-07-12 重设计 v2）每个工具卡包含 5 个元素：名称、tagline、chip 式标签、难度星级、CTA "进入 →"。用户反馈信息密度过高，视觉负担重。

本次设计围绕"简化"展开，提供两档可切换的卡片模式。

## 2. 设计决策速览

| 项目 | 决定 |
|------|------|
| 首页默认模式 | **简洁（Simple）** — 仅名称 + 一行描述 |
| 首页可切换至 | **详细（Detail）** — 名称 + 描述 + 纯文字 tags + 星级 |
| 切换开关 | 分类标签栏右侧外部，文字切换 |
| 偏好持久化 | `wx.setStorageSync('cardDisplayMode', 'simple'\|'detail')` |
| 默认值 | `simple` |
| 切换范围 | 整页（全部视图 + 各分类视图）统一切换 |
| tools-all 页 | 固定详细模式，无切换开关 |

## 3. 首页卡片结构

### 3.1 简洁模式（Simple · 默认）

```
┌───────────────────────┐
│  子网计算器             │  名称 30rpx, weight 500, #141413
│  IP 地址到底怎么算？     │  tagline 20rpx, #6c6a64, 1 行截断
└───────────────────────┘
```

- **去掉**：tags chip、难度星级、CTA "进入 →"
- **去掉**：`.tool-meta-row` 整个区块
- **卡片内边距**：从 `28rpx 24rpx 20rpx` 统一为 `24rpx`（各边等距）
- **点击**：整卡可点，保留 `.tool-hover` 反馈
- **tagline 不存在时**：fallback 到 `description` 字段

### 3.2 详细模式（Detail · 用户切换）

```
┌───────────────────────────┐
│  子网计算器                 │
│  IP 地址到底怎么算？         │
│  #可视化 #交互式  ★★☆       │  tags 纯文字 + 星级
└───────────────────────────┘
```

- **tags**：去掉 `border` + `background` 样式，纯 `#6c6a64` 文本，空格分隔，字号 `18rpx`
- **难度**：星级符号（★☆☆ / ★★☆ / ★★★），字号 `18rpx`，颜色 `#8e8b82`
- **tags 与难度之间**：空格分隔，无额外符号
- **CTA "进入 →"**：不展示（整卡可点）
- **无 tags 时**：仅展示星级（或无）
- **无 difficulty 时**：仅展示 tags（或无）

### 3.3 预告卡片（即将上线）

两种模式下表现一致，不变：

```
┌───────────────────────┐
│  TLS 动画机             │
│  TLS 1.3 握手 · 证书链   │
│  即将上线                 │
└───────────────────────┘
```

- 保持 opacity `0.45` 半透明
- 内容：名称 + tagline（fallback description）+ "即将上线"标签
- 不可点击（保持现有行为）

## 4. 切换开关

### 4.1 位置与样式

```
[全部] [计算机网络] [操作系统] [密码学] ...  [简洁][详细]
                                              ↑scroll-view 外部右侧
```

- 放在 `<scroll-view class="category-tabs">` **外部右侧**，不随分类滚动
- **胶囊式 segmented control**：两个按钮「简洁」「详细」连为一体
- 边框：`2rpx solid #d0ccc0`，圆角 `14rpx`
- 活动态：背景 `#cc785c`，文字白色
- 非活动态：背景透明，文字 `#6c6a64`
- 字号 `22rpx`，weight `500`，内边距 `8rpx 20rpx`

### 4.2 交互行为

- 点击"简洁"→ 切换到简洁模式，"简洁"变珊瑚色
- 点击"详细"→ 切换到详细模式，"详细"变珊瑚色
- 切换瞬间更新所有卡片，**无过渡动画**
- 触发 `setData` 更新卡片渲染

### 4.3 持久化

- 存储 key：`cardDisplayMode`
- 取值：`'simple'` | `'detail'`
- 读取时机：`onShow()` 时读取
- 写入时机：每次切换时写入

## 5. tools-all 页

### 5.1 卡片结构（固定详细模式）

```
┌───────────────────────────┐
│  子网计算器                 │
│  IP 地址到底怎么算？         │
│  #可视化 #交互式  ★★☆       │
└───────────────────────────┘
```

- 与首页详细模式完全一致
- 可用工具 + 不可用工具（半透明+"即将上线"标签）在同一网格
- **不加切换开关**
- 去掉 CTA "进入 →"（同首页详细模式）

## 6. 涉及文件

| 文件 | 改动内容 |
|------|----------|
| `pages/index/index.wxml` | 卡片模板简化；添加切换开关 |
| `pages/index/index.wxss` | 移除 chip 样式；更新卡片间距；添加开关样式 |
| `pages/index/index.js` | 添加 `cardMode` 状态；切换逻辑；持久化读写 |
| `pages/tools-all/tools-all.wxml` | 卡片模板简化（去 CTA、去 chip） |
| `pages/tools-all/tools-all.wxss` | 移除 chip 样式；更新卡片样式 |
| `pages/tools-all/tools-all.js` | 无需改动（已有 tagline/description fallback） |

## 7. 不做的范围

- 不改动 `utils/tool-registry.js` 数据模型
- 不改动工具详情/intro 页面
- 不改动入场动画（fadeSlideIn / fadeSlideUp）系统
- 不改动 tools-all 的标签栏和分类视图逻辑
- 不添加新图标、emoji 或其他视觉元素

## 8. 边界情况

| 场景 | 处理 |
|------|------|
| 工具有 tagline 无 tags/difficulty | 简洁/详细都只显示名称+tagline |
| 工具有 tags 无 difficulty | 详细模式只显示 tags |
| 工具有 difficulty 无 tags | 详细模式只显示星级 |
| 两者都没有 | 仅显示名称（极罕见，仅完全未配置的工具） |
| 存储中无偏好值 | 默认 `simple` |
| 存储值异常（如拼写错误） | fallback 到 `simple` |

## 9. 设计约束

遵循 Claude Design 暖奶油画布规范：

- 页面背景 `#faf9f5`
- 卡片背景 `#efe9de`，圆角 24rpx
- 主色 `#cc785c`（开关激活态）
- 正文字色 `#141413`
- 次要文字 `#6c6a64`
- 零阴影
