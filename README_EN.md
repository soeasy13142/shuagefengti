# Shua Ge Feng Ti

[中文](README.md)

A WeChat Mini Program learning toolbox. Started as a local quiz practice tool, then gradually picked up sorting visualization, a subnet calculator, TCP state machine animation, data structure demos, and a learning dashboard. It's become a bit of a Swiss Army knife for studying.

Everything runs locally — no backend, no accounts. The questions you import, your quiz history, your wrong-answer book — it all lives on your own device.

## What It Does

### Quiz Practice

The original module, and still the core. Import questions from Markdown files; the parser auto-detects five types: single-choice, multi-choice, true/false, fill-in-the-blank, and short answer. The parser is built with nothing but regex — no Markdown libraries — because every extra dependency costs precious kilobytes in a Mini Program's size budget.

Two modes: Practice mode shows you the explanation immediately after each question (good for learning as you go); Exam mode mimics a real test — no feedback until you submit everything. Wrong answers automatically land in a dedicated review section. Every session is saved with timestamps, accuracy, and per-question detail.

### Subnet Calculator

Probably the nerdiest tool in the box. Enter an IP and CIDR prefix, get subnet mask, network address, broadcast address, and usable host range in real time.

The part I actually enjoyed building is the 32-bit binary visualization. Thirty-two little cells in a row — network bits in coral, host bits in light blue — four rows showing the IP, mask, network, and broadcast address in binary, with a clear demarcation between network and host portions. The idea is simple: a lot of people get stuck on the binary-to-decimal conversion when learning subnetting. Seeing the bits directly makes the concept click faster.

Later I added a step-by-step AND operation animation. You can play through it byte-by-byte (4 steps) or bit-by-bit (32 steps), with each step highlighting the two bits being ANDed and the result. The animation sequence is generated by a pure function `generateAndSteps()` in `utils/subnet.js`, completely decoupled from the UI, so it has its own dedicated tests. Forty-three test cases cover everything from /0 to /32, including edge cases like /31 point-to-point links.

### TCP Animator

TCP state transitions aren't complicated in theory, but reading about them in prose is a good way to fall asleep. This tool turns the TCP state machine into a visual, interactive diagram. A step-by-step animation walks through the three-way handshake and four-way teardown, with each packet (SYN, SYN-ACK, ACK, FIN, etc.) triggering a state change with visual highlighting.

The state machine definition lives entirely in `utils/tcp-states.js` as a pure data table. The animation engine just reads this table and drives the UI — it knows nothing about TCP itself. Swap in a different state machine table and you've got a different protocol animator.

### Data Structure Visualization

Interactive demos for four data structures: binary search trees (insert, delete, three traversal modes), stacks and queues (real-time push/pop/enqueue/dequeue animation), hash tables (collision resolution with chaining, visualized), and graph BFS/DFS search.

Each data structure's core algorithm is extracted into `utils/` (`bst.js`, `graph.js`, `hash-table.js`). The pages only call the algorithms, get back step sequences, and render them. This separation means the algorithms are independently testable, and changing the UI style never touches the logic.

### Sorting Visualization

Three sorting algorithms (bubble, selection, quick) rendered as animated bar charts. The user enters a sequence of numbers, and each comparison and swap is color-coded on the bars — compare, swap, pivot, sorted, each with its own visual state. Step counter, comparison counter, swap counter, playback controls, speed adjustment, all there.

### Learning Dashboard

Pulls quiz records and wrong-answer data into a data analysis page. Four KPI cards at the top (total questions, sessions, average accuracy, wrong answers), a bar chart of accuracy by question type, a 7-day activity trend, and a set of "smart" suggestions at the bottom. The suggestions read a bit like AI output, but they're actually generated by a pure rule engine — e.g., "multi-choice accuracy below 60%" triggers a targeted practice suggestion. No remote API calls.

## Design

The visual language is warm cream + coral. Page background `#faf9f5`, cards `#efe9de`, CTA buttons `#cc785c` coral. Zero shadows — all depth comes from contrast between color blocks. This was a deliberate choice: in WeChat's rendering environment, it's cleaner than box-shadow, and avoids shadow rendering bugs on certain Android devices.

Headings use Georgia serif, body text the system default sans-serif. No custom font files. The entire design system runs on a handful of CSS variables — change one, and it propagates everywhere.

## Under the Hood

Native WeChat Mini Program framework (WXML + WXSS + JS). No third-party UI libraries, no build toolchain beyond what WeChat DevTools provides.

Storage is `wx.setStorageSync` — JSON serialization straight to local storage. One gotcha: URL parameters have a length limit, and passing question data between pages would overflow. The fix: a `tempImportData` key in `utils/storage.js`. Previous page writes, next page reads, data gets cleared after use.

Testing is Jest. Eleven suites, 218 test cases, all green. Every utility function and algorithm has test coverage. Page-level WXML/WXSS isn't tested — the ROI on end-to-end testing in a Mini Program environment isn't great. Extract the logic, test that, and trust the rendering.

## Quick Start

WeChat DevTools and Node.js ≥ 16 required.

```bash
npm install
npm test
```

Then open the project root in WeChat DevTools.

To import questions: prepare a Markdown file (see `test-questions.md` for format), tap "Start Quiz" → "Import Paper", select your file.
